# 判断提示的外部经验适配边界

本 Skill 独立改写并吸收了以下提示结构；不复制原始 Prompt、角色口吻、论文措辞、图表、代码、参数或结论。

| 来源 | 保留的判断结构 | 未采用的部分 |
| --- | --- | --- |
| XiaoMa 建模/编程/论文角色 | 先定义子问题应回答的结论；运行必须用真实输出；异常回传上游；官方模板优先 | 固定两模型、固定图类、固定 DOCX 交付 |
| ZSL 建模手 | 赛制和数据条件会改变建模取舍；先审计附件；记录方法前提与验证 | 固定模型数量、固定“必做”分析、算法清单直接选型 |
| LocalWorkflow 角色流程 | 阶段边界、实现前输入充分性、真实运行结果 | 只以自然语言报告作为交接契约 |
| Nature Writing | 一句话论证、模块动机、主张-证据-边界、怀疑式自审 | 期刊新颖性门槛、固定基线/消融和投稿导向语气 |
| Nature Figure | 先定义图的结论、证据逻辑和审稿风险，再作图 | Python/R 二选一的交互门禁；本 Skill 保留 Python/MATLAB/R 选择规则 |

这些提示只有在能影响当前问题定义、候选选择、验证、人工决策或论文主张时才加载和回答。它们不构成额外必交产物，也不要求每题都有创新、对照或复杂模型。
