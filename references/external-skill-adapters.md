# External Skill Reference Adapters

This skill may refer to installed mathematical-modeling and academic-writing skills, but that reference is optional calibration, not a prerequisite. The problem statement, official rules, upstream handoff package, and verifiable evidence always take priority.

Paths, versions, licensing, and availability can change. Before copying, modifying, or bundling any external asset, verify its repository LICENSE, NOTICE, and competition rules. This register is not a license grant.

## Use order

1. Establish the judgment from the active problem structure, data, and official rules.
2. Load one external source only when a concrete gap exists, such as competition format, an isomorphic case, or a delivery tool.
3. Rewrite extracted material as a current-task structured record, including source, location, transferable part, and non-transferable part.
4. Re-validate with this problem's data, model, and run results. Never import external numbers, parameters, or conclusions as evidence.

## Never do

- Rename and reuse wording, figures, tables, code, parameters, or conclusions from outstanding papers.
- Treat an award in another problem as proof that a method fits this one.
- Copy code, templates, or images whose license or attribution obligations are unclear.
- Quietly alter this problem's objective, constraints, variable meanings, or result metrics to fit an outside asset.

For task-level procedures, read the appropriate module adapter reference.
