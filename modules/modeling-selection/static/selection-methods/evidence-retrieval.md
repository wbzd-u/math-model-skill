# 第二轮：结构化证据检索

## 目标

寻找能约束候选路线的真实证据，而不是搜集题目背景相似但结构无关的材料。

## 检索键

先为每个子问题形成结构查询，不以题目名称作为唯一检索词：

```text
任务角色 + 状态/事件结构 + 目标与硬约束
+ 数据规模/粒度 + 不确定性 + 必交输出 + 验证要求
```

优先使用 `task_tags`、`difficulty_drivers`、变量角色、数据状态和输出类型查询 `references/index.yaml`。知识库后续加入历史赛题、优秀论文或方法卡时，仍使用同一检索键。

## 必须执行

1. 分别检索 `case_cards`、`method_cards` 和 `validation_cards`。
2. 对每个命中项核对原始来源或可定位卡片，不只阅读标题和摘要。
3. 记录来源类型、结构相似点、适用条件、当前题可迁移部分、不可迁移部分和核验状态。
4. 区分证据强度：官方规则或定义、理论/方法条件、结构相似案例、经验性案例、仅背景相关材料。
5. 没有命中时写 `knowledge_status: none-available`；只核验部分来源时写 `partial`。
6. 未实际读取的论文、网页、历史题或方法卡不得登记为已验证证据。

## 提示问题

- 相似的是数学结构、数据条件和评价口径，还是只有应用背景？
- 历史论文的方法依赖哪些本题不具备的数据、假设或规模条件？
- 该证据支持“方法可用”，还是只说明“曾有人使用”？
- 哪个理论条件或失败案例能直接淘汰候选路线？
- 本题有哪些结构没有任何现成证据，需要保守设计和更强验证？

## 输出

填写 `candidate-methods.yaml.evidence_records` 与 `knowledge_status`。每条证据使用稳定 `ev-*` ID，候选生成后只引用这些已登记记录。

## 失败信号

- 仅因题目背景相同就认定方法可迁移。
- 把优秀论文的结论或参数值直接当作本题事实。
- 声称“历年常用”“文献证明”却没有可定位来源。
- 知识库为空时伪造案例，或因没有案例就停止结构推理。
