# 方法选择与模型设计工作流

1. 校验上游版本、状态、文件哈希和选择范围。
2. 对每个选定子问题重申必交输出、成功判定、验收测试、数据状态和结构难点。
3. 按结构标签检索案例卡、方法卡和验证卡；记录没有命中的知识缺口。
4. 先生成一个最小可行方法链；只有结构、假设、信息利用或风险处理实质不同才增加候选。
5. 使用相同维度比较候选，不强制虚假的加权总分。
6. 有依据地确定当前主方案；按实际需要增加基线或备选，不为格式凑模型。
7. 把选定路线规格化为符号、派生量、假设、组件、关系、目标、约束和组件依赖。
8. 为主要结论、硬约束、高风险假设和关键结构难点设计可证伪验证。
9. 必要时生成诊断性试算契约，用最小成本检验可识别性、可行性或结构假说；根据 solver 反馈迭代。
10. 生成正式实施契约，声明输入、阶段、输出、不可修改项、solver 自主范围和回退条件。
11. 日常运行 `explore` 校验；正式求解前运行 `handoff` 校验。

若任何阶段发现题意、核心变量或数据定义错误，返回 `problem-analysis`。若只是候选方法前提不满足，在本模块更换或降级候选路线并记录原因。
