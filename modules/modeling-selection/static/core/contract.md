# 方法选择与模型设计契约

## 目标

把已通过题目分析门禁的问题定义转化为有证据、可比较、可证伪、可实现的模型路线和 solver 契约。

## 强制原则

1. **上游只读**：所有事实、要求、变量角色、成功判定和阻断项以 `analysis/` 为准。
2. **先能力后名称**：先说明候选链必须表示、估计、决策、聚合或验证什么，再加载具体方法卡。
3. **证据与迁移限制并列**：历史案例和方法证据同时记录相似点、适用前提与不可迁移部分。
4. **候选可证伪**：每条候选路线必须有数据前提、失败信号、验证方式和拒绝条件。
5. **选择可解释**：主方案必须有理由；基线、备选和额外候选只在能提供真实比较或风险缓解时加入，不使用数量或复杂度冒充质量。
6. **模型规格完整**：按组件性质明确实际存在的变量、关系、目标、约束、参数来源、输入输出和模型链；不存在的类别保留空数组，不填写“无”来凑完整。
7. **假设不升级为事实**：新增假设独立登记，并说明依据、影响和验证方法。
8. **solver 边界明确**：solver 可选择语言、库和数值实现，但不得改变目标、硬约束、核心假设或评价口径。
9. **允许迭代**：探索方案可以通过低成本诊断性试算被否定和修订；试算结果属于 solver 证据，不得预写为当前结论。

灵敏度、误差分析和可视化属于验证或呈现，不计作独立模型。共享同一机制、变量和目标的不同精度展开属于一个模型族。
