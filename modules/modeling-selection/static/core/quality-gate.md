# 方法选择与模型设计科学检查

## handoff 必须满足

- 上游范围和文件哈希仍然有效，选定子问题属于 `handoff_allowed_subproblem_ids`。
- 主方案覆盖选定子问题及题目要求，没有改变成功判定、硬约束或评价口径。
- 数据、参数来源和关键假设足以支持实现；不确定内容明确标为假设或限制。
- 模型规格中的变量、关系、目标、约束和组件输入输出一致且可追溯。
- 主模型组件、主要科学主张、硬约束和高风险结构难点有可证伪验证。
- 实施契约没有引入规格中不存在的目标、变量、约束或假设。
- 不可修改语义已进入实施契约，主模型组件及其验证被实际调度。
- solver 的自主范围和必须反馈的触发条件明确。

## 只警告，不阻断

- 候选只有一个，或多于四个。
- 没有基线、备选或完整比较矩阵。
- 候选角色部分重叠，但作用和简化关系说得清楚。
- 模型链存在有意的迭代或反馈环，并已说明收敛或终止条件。
- 非关键组件尚未验证，部分审计维度未运行，或覆盖指标不是 1.000。
- 知识库没有命中，但已如实记录 `none-available` 或尚未检索状态。

任何模式都不得伪造历史案例、文献、数据、模拟结果或预期性能。不得为了通过形式检查强制添加独立模型。
