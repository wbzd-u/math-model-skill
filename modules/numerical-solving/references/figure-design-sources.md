# 图表设计来源与筛选边界

本规则由以下材料交叉筛选而成：

- `nature-figure`：采用“结论 -> 证据链 -> 面板/编码 -> 审查风险”的图表契约、颜色语义一致、数据与统计可追溯、导出后视觉 QA。只作为高质量科学图的一般实践，不采用其 Nature 期刊尺寸、显微图、图像处理或复杂拼图规则。
- `nature-figure` 的后端选择与单一渲染链：采用“显式选择后端、不要为了预览悄悄换后端、跨语言时通过干净数据交接”的可复现原则。扩展为数模的 Python、MATLAB、R 三种后端；不继承其仅在 Python/R 间持久化用户偏好的交互规则。
- `math-modeling-xiaomacoltai` 的《可视化规范》：采用数据/过程/结果三类图的区分、坐标与单位、色彩可达性、SVG 与 300 DPI PNG 导出、统计量由当前代码生成。其“默认无网格”等表现规则降级为可覆盖的默认风格。
- 本机优秀论文样本：`RGV 的动态调度优化问题`、`高温作业服设计`、`大型百货商场会员画像描述`。阅读其图表用途时关注了流程/状态结构、约束或方案比较、动态响应与数据趋势。样本用于观察图表在建模论文中的论证角色，不作为质量结论、固定图型或硬要求。

禁止复制来源中的图、代码、图注、具体配色、参数或结论。当前题的官方规则、可用数据、模型语义和真实 solver 输出始终优先。
