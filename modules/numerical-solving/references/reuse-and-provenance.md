# 复用、许可与来源记录

## 1. 基本判断

“公开”不等于“可复制”。数学方法、论文正文、论文附带代码、开放仓库和库 API 使用不同的许可与引用边界。先遵守比赛或项目对外部资源和 AI 使用的规则；规则不允许时立即说明限制。

## 2. 来源类型

### 成熟库公开 API

优先调用已安装且维护良好的公开 API。记录包名、版本、官方文档、包仓库、许可证和用途。除非确有必要，不复制其内部实现。

### 开源代码

复制、修改或 vendoring 前核对仓库自己的许可证，而不是只看论文或项目主页。记录仓库、commit、原文件/行定位、本地路径、修改、署名和 LICENSE/NOTICE 义务。无许可证或许可证不明时不复制代码。

### 论文方法

从原始论文的公式、章节或算法描述独立组织实现，记录 DOI/稳定链接、方法定位、迁移部分和偏离点。重新确定适合本题的数据、参数和验证方式。不得复制论文文字、图表、结果，也不得把已发表参数自动当作本题事实。

论文附带代码仍按“开源代码”单独核对许可证。引用论文不等于取得代码许可。

### 伪代码、博客和问答

伪代码可以作为算法语义参考，但不要逐行翻译或只改变量名。博客、问答和搜索摘要优先用于定位官方文档或原始论文；来源和许可证不清的代码片段不得复制。

## 3. 复用记录

对影响结果的每个来源，在 `implementation-provenance.yaml` 中记录：

- 稳定 ID、类型、名称、URL/DOI、版本或 commit；
- 许可证标识、证据位置、兼容性和义务；
- 关联 `cmp-*`、`run-*`、本地源码和用途；
- 使用方式：API 调用、改编代码、独立论文实现、基准或概念参考；
- 本地改动、语义偏差、署名位置和验证测试。

仅在许可证要求时生成 `THIRD_PARTY_NOTICES.md`，不要机械创建。

## 4. 自研条件

以下任一情况即可自研：没有语义匹配实现；许可证不明确或不兼容；环境或依赖代价不可接受；现有实现缺少关键约束、诊断或精度；题目需要新的组合结构；本题特有逻辑很短且自写更透明。

记录现有方案为何不适配。自研非平凡内核至少使用手算/解析小例、性质测试、成熟实现或公开 benchmark 中的一种进行验证。不要把独立实现自动宣称为原创算法。

## 5. 门禁

始终阻断：无许可复制源码、未履行署名义务、通过重写掩盖来源、把外部结果当本题结果、为适配库静默改变模型语义。

probe 可暂时警告：依赖未锁版本、论文尚未核验原文、重要软件引用尚未整理。solve 前必须完成所有实际影响结果的来源、版本、语义映射、许可和验证记录。
