# 数值求解质量门禁

## 始终阻断

- 上游快照或当前原始输入哈希失效且未重新准入；
- 结构化文件不可解析、稳定 ID 冲突、引用不存在或路径越出项目；
- 静默改变不可变语义，或对未解析字段、单位和参数进行猜测；
- 声称结果来自未执行的代码，伪造日志、哈希、指标或成功状态；
- 复制、修改或内嵌许可证未知/不兼容的第三方源码；
- 把论文、示例或公开数据中的结果当作本题运行结果。

## `probe` 完成条件

- 有明确诊断问题、组件或验证测试；
- 至少一次真实运行或一个可复现的阻断证据；
- 简化、失败信号、证据和下一步已记录；
- `ready_for_writing` 为 `false`。

诊断否定模型仍可标记 probe 完成，同时生成反馈。

## `solve` 交接条件

- 上游契约为 `READY`，快照和输入完整；
- 契约要求的执行阶段和必需输出均有真实运行证据；
- 每个契约验证测试都有结果；需要修订模型的失败不得被降格为限制；
- 关键硬约束、不变量、终止或收敛状态有证据；
- 外部实现的版本、来源、许可、用途和本地改动可追溯；
- 命令、环境、输入、源码、配置、seed、日志和输出可复现；
- 结果主张绑定证据并公开限制。

没有基线、备选、敏感性分析、固定数量图表或特定语言本身不构成失败，除非契约明确要求。
