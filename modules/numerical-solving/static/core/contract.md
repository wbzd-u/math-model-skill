# 数值求解契约

## 目标

把已批准的数学模型转化为真实运行、可检查、可复现的结果，同时用诊断证据发现模型或数据问题。

## 权威来源

1. 题意、字段和单位以 `analysis/` 为准。
2. 模型、假设、目标、约束、参数政策和验证要求以 `modeling/` 为准。
3. `solver/intake.yaml` 只是上述来源的运行时绑定，不得反向覆盖它们。
4. 原始数据只读；所有派生数据和结果写入 `solver/`。

## 强制原则

- 先验证上游哈希，再写代码或运行实验。
- 每个执行阶段回链到 `run-*`、`cmp-*`、输入引用、输出引用和 `val-*`。
- 只把真实命令产生的文件登记为结果；保留失败运行和日志。
- 不猜字段、单位、参数或缺失值政策。
- 不静默改变不可变语义；需要改变时生成结构化反馈。
- 复用成熟实现优先，但外部实现必须语义匹配、来源可追溯且许可允许。
- probe 结果只回答诊断问题；solve 结果才可能进入论文。
- 若 probe 暴露出需要改变核心假设、目标取舍、外部数据政策或正式主路线的选择，生成上游反馈并更新共享人机决策台账；在人工确认前，不得用替代实现冒充原 `solve` 契约。
