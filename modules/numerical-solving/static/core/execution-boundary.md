# 执行边界

## Solver 可自主决定

- 语言、包、等价求解器和公开 API；
- 模块结构、数据结构、缓存、并行化和日志；
- 不改变语义的缩放、向量化和数值稳定化；
- 在契约授权范围内的随机种子、容差、停止准则和求解预算；
- 输出文件的内部格式和辅助诊断图。

所有决定写入 `run-plan.yaml`，重要依赖写入来源台账。

## 必须反馈

- 改变目标、硬约束、变量角色、变量域、单位、核心假设或评价口径；
- 改变不允许校准的参数或把外部论文参数直接套入本题；
- 把全局最优要求降为启发式可行解，或改变统计推断/预测声明；
- 删除数据、约束或失败结果以改善表现；
- 正式运行依赖未解决的字段或参数猜测。

`probe` 可以临时简化规模、精度或非核心数值设置，但要记录简化和结论边界。诊断性松弛不得进入正式结果。

## 语义偏差登记

实现中发现任何可能改变结论含义的差异，立即写入 `run-plan.yaml.semantic_deviations`。未获得上游修订前，将其状态设为 `feedback-required`，停止受影响的正式阶段。
