# 复用政策

## 决策原则

先判断语义匹配，再判断是否复用。通常优先成熟库 API、许可明确的官方或作者实现、原始论文的独立实现，最后才是自研。

- 不为降低依赖而复制成熟库内部实现。
- 不把“公开可见”当作“允许复制”。
- 不把引用论文当作取得附带代码许可证。
- 不把库默认值当作模型参数依据。
- 不因复用而降低验证强度。

## 最低记录

对影响结果的外部来源记录名称、稳定 URL 或 DOI、版本或 commit、许可证证据、用途、关联组件、本地改动、署名位置和验证方法。

对自研非平凡数值内核记录现有方案为何不适配，以及小例、解析解、成熟实现或公开基准中的至少一种正确性检查。

详细规则见 `references/reuse-and-provenance.md`。
