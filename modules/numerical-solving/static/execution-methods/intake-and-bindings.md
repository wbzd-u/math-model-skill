# 第一轮：准入与绑定

## 目标

把上游稳定 ID 绑定到可执行数据、参数、阶段和输出，不通过猜测补洞。

## 提示

1. 说明本次是 `probe` 还是 `solve`，以及要回答的诊断问题或正式范围。
2. 检查契约状态和三项快照哈希；读取契约引用的模型规格与验证计划。
3. 从数据清单、变量映射和数据任务矩阵解析 `data-*`、`var-*`、`param-*` 和前序阶段产物。
4. 对每个 `run-*` 写出输入来自文件、字段、固定参数、校准结果还是前序输出。
5. 检查文件存在性、格式、字段、单位、规模、只读属性和当前 SHA-256。
6. 无法唯一解析时写入 `unresolved_items`。probe 只可绕开与诊断无关的项；solve 不得执行受影响阶段。

不要因为文件能打开就假设字段含义正确，也不要把预处理后的文件覆盖原始附件。
