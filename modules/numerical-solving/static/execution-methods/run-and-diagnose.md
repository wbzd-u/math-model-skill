# 第四轮：真实运行与诊断

## 目标

让每个数值结果都能回到一次可复现运行，并保留失败信息。

## 提示

1. 每个配置或情景建立一个新的 `exp-*`，执行登记的真实命令。
2. 记录工作目录、输入和配置哈希、源码状态、seed、开始结束时间、退出码、日志和输出哈希。
3. 保存关键中间量、迭代历史、约束残差、终止原因、求解器状态和运行时间。
4. probe 只运行回答诊断问题所需的最小实验，并明确简化不能支持哪些结论。
5. solve 执行契约要求的阶段、输出和验证，不因结果不理想而删掉失败运行。
6. 超时先检查复杂度、剖析瓶颈、缩小 probe 或调整等价实现；不要只盲目延长等待。

失败分类为实现、环境、数值、数据或契约模型问题，并记录下一步和最小复现证据。
