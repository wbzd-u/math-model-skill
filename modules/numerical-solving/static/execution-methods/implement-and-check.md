# 第三轮：实现与局部校验

## 目标

先证明代码忠实实现模型，再进行昂贵的全量运行。

## 提示

1. 建立一个从输入到一个关键输出的最小可运行链。
2. 保持代码变量、单位和组件边界可回链到模型规格；在配置中暴露参数、seed、容差和预算。
3. 对输入解析、单位转换、目标/约束计算和关键中间量编写小型检查。
4. 按模型选择合适的正确性证据：手算小例、解析解、性质测试、边界案例、成熟库对照或公开 benchmark。
5. 检查数据泄漏、索引错位、维度广播、符号方向、单位换算、时间顺序和随机性。
6. 复用库也要检查返回状态、终止原因、约束残差和默认值；不得只看“没有报错”。

局部校验失败属于实现问题时修复；指向模型矛盾时停止扩大实验并反馈。
