# 图表设计：以判断为中心，而不是以图形为中心

在编写绘图代码前，先判断是否需要图。图不是固定交付物；只有它能更清楚地支持结论、检验模型、暴露风险或传达决策时才生成。不要因为某种图常见、某篇优秀论文画过、或需要填充版面而作图。

## 绘图后端：Python、MATLAB 或 R

在 `solver/run-plan.yaml.plotting_backend` 中记录最终绘图后端、理由和跨语言数据交接路径。先按以下顺序决定：

1. 用户、当届模板或已有项目代码已指定语言时，遵守该选择。
2. 求解代码已有稳定语言时，默认用同一语言作图，以减少数据搬运和复现断点。
3. 只有当另一语言提供明确且必要的能力时才跨语言：将已清洗、稳定列名的 CSV/TSV 或其他可追溯数据写入 `data_handoff_path`，最终图的绘制、预览、导出和视觉 QA 全部使用同一个绘图后端。
4. 还没有实现语言时，按当前工作负担选择，而不是按“高级程度”选择：
   - **Python**：通用数据处理、优化/仿真结果、定制坐标与布局、地理或网络扩展，使用 `matplotlib` 为主，按需使用 `seaborn`。
   - **MATLAB**：求解器、控制/信号/矩阵计算或既有代码已在 MATLAB，使用 `tiledlayout` 和 `exportgraphics` 保持同源输出。
   - **R**：统计推断、回归诊断、分布/分组比较或既有 R 数据工作流占主导，使用 `ggplot2` 及必要的统计包。

没有图表需求时记录 `not-needed`。不要因美观跨语言；也不要让一种语言生成数据、另一种语言临时绘图却不记录交接文件和命令。

## 图表契约

每幅候选图先在 `solver/solver-report.md` 或运行计划中记录以下简短图卡，再绘图：

```text
结论：这幅图要支持或否定的本题判断。
问题：它帮助读者作出什么比较、诊断或决策？
数据：真实 exp-* 输出、数据字段或已注册中间量。
编码：x、y、系列/分组、区间或约束标记各表示什么。
比较：基线、情景、反事实或无；不制造比较对象。
风险：可能误导的尺度、缺失、抽样误差、不可比性或选择偏差。
产物：solver/figures/ 中的文件及其关联的 claim 或验证。
```

结论必须包含可检验的动词，例如“主方案在相同约束下减少等待时间”，而不是“结果展示”。每个面板承担独特证据；去掉某面板不影响判断时，应删除或合并它。主结论使用最清楚的面板或区域，补充诊断不与主证据争夺视觉权重。

## 按问题选择图形

| 要回答的判断 | 首选图形 | 需要同时检查 |
| --- | --- | --- |
| 数据是否有偏态、异常、缺失或分组差异 | 分布图、ECDF、箱线图、散点图、缺失模式图 | 样本量、变换、异常处理规则；不要只报均值 |
| 预测是否在未见时段/样本上可靠 | 观测-预测叠图、滚动误差图、残差诊断、校准图 | 训练/验证划分、时间泄漏、区间定义和基线 |
| 优化方案是否更好且可行 | 同尺度方案比较、Pareto 前沿、瓶颈/约束余量图 | 相同约束、目标口径、可行性、最优性或求解间隙 |
| 结论是否对参数或情景稳定 | 参数响应曲线、情景矩阵、翻转边界、tornado 图 | 扰动范围依据、其他参数固定方式、决策是否真的翻转 |
| 仿真机制或动态过程是否可信 | 状态轨迹、情景带、重复运行分布、流量/守恒图 | 初值、随机种子、重复次数、收敛或守恒误差 |
| 排名/评价是否稳健 | 排名变化、权重扰动、指标贡献、支配关系图 | 指标定义、权重来源、并列与排名翻转 |
| 分类、聚类或识别是否真正服务任务 | 混淆矩阵、错误案例、嵌入/分群图、下游误差传播 | 类别不平衡、划分方式、特征变换和下游决策影响 |
| 路径、空间或网络决策有何差异 | 地图/轨迹、流量、关键节点或干预前后比较 | 坐标系、比例、边/节点语义；不要只画装饰性网络图 |

图形类型只是候选。雷达图、饼图、三维柱图、收敛曲线和流程图必须先证明比表格、散点、区间或直接比较更能回答当前判断；算法收敛图本身不能证明方案质量。

## 视觉编码与颜色

- 在同一项目中保持方法、情景和状态的颜色语义一致。优先使用一个中性系列、一个主信号系列和至多一个风险/强调系列；官方模板或用户配色优先。
- 基线宜用中性灰或低饱和色，主方案用稳定主色；风险、违约或负向变化可用橙红色。红绿不能作为唯一编码，必须同时使用线型、点形、纹理、直接标签或清晰文字。
- 连续且有顺序的量使用感知连续的顺序色；只有围绕有意义基准（如零、目标或平均值）比较高低时才使用发散色。不要使用彩虹色盘制造边界。
- 类别少且位置稳定时优先直接标注，类别多或位置变化时使用简洁图例。不要用重复图例或大面积深色填充遮挡数据。
- 默认白色背景。网格线不是禁令，但仅保留能帮助读取明确数值的轻量参考线；零线、阈值、容量线或事件线必须解释其数值含义。

## 数值完整性与导出

- 标题、坐标轴、单位、范围、图例、数据变换和比较口径必须与模型及论文术语一致。不得通过截断坐标轴、选择性筛样或未说明的平滑夸大差异。
- 区间、误差条、p 值、显著性、置信带和汇总统计必须由当前运行计算，并说明其统计含义；标准差不能冒充置信区间。
- 定量图优先输出适合 LaTeX 的 SVG 或 PDF；需要栅格预览或官方要求时再输出至少 300 DPI PNG。文本应在矢量文件中保持可编辑，除非符号兼容性迫使转换。
- 导出后检查最终插入尺寸：文字可读、标签不重叠、颜色可区分、图例不遮挡、同尺度比较真正同尺度。论文只插入与 `results-manifest.yaml` 和真实文件相互可追溯的图。

## 不能迁移的做法

不要将 Nature 的期刊尺寸、显微图完整性清单、复杂多面板拼图或固定字体当作数模竞赛要求。不要从优秀论文复制图形、配色、参数或结论。优秀论文样本只能启发“图为何服务某个模型判断”，不能规定当前题必须画同一种图。
