# 执行与验证判断提示

在正式运行、比较结果或制作论文图之前，使用这些问题把“程序能运行”与“结论可信”分开。

1. 哪个不变量、已知极限、边界样例、守恒关系或小规模穷举可以最快否定当前实现？它是否已写入 `invariant_checks` 或 `val-*`？
2. 当前比较是否公平：相同输入、信息截止点、目标口径、硬约束、计算预算和评价单位是否一致？若不一致，图表或指标还能支持什么结论？
3. 若结果异常、不稳定或不可行，证据更支持代码/依赖错误、数值控制问题、数据语义问题还是模型假设失效？分别需要什么日志或对照才能区分？
4. 预测、分类、前后比较或重复测量是否在主体、组或时间边界上验证？任何看似提升的结果是否可能来自泄漏、选择性汇报或不一致的预处理？
5. 哪项诊断最可能改变决策，而不是只展示一个运行过程？若结果不足以支持主张，应降低主张、补充验证或反馈上游。

这些提示不要求统一跑分、消融或多次随机试验。只执行与当前模型族、风险和主张有关的诊断。
