# 第二轮：复用决策与运行计划

## 目标

用最小、可靠、语义匹配的技术栈实现契约，而不是展示算法数量或代码复杂度。

## 提示

1. 对每个非平凡 `cmp-*` 提取必须保持的目标、约束、变量域、精度和诊断能力。
2. 依次检查成熟库 API、官方/作者实现、原始论文方法和自研方案；显然匹配时一个候选即可。
3. 核对版本、维护状态、平台可用性、许可证、竞赛规则、规模适配和可复现性。
4. 将采用和拒绝的关键来源写入 `implementation-provenance.yaml`；不得伪造搜索或许可结论。
5. 在 `run-plan.yaml` 写明语言、依赖、阶段输入输出、源码、命令、seed、容差、停止条件、不变量检查和验证映射。
6. 若采用近似、松弛或替代算法，判断其是否仍满足契约语义；不满足时生成反馈。

本题特有的模型组装和业务逻辑可以自行实现；成熟的数值内核优先调用库。
