# 第五轮：验证与交付

## 目标

把运行证据转化为可审查的测试结果、输出清单和有限度结论。

## 提示

1. 对每个契约 `val-*` 记录实际过程、指标、判据、状态和证据，不执行时说明原因。
2. 按模型族补充最少的必要诊断，例如优化可行性和 gap、ODE 网格收敛、仿真 Monte Carlo 误差、预测泄漏与切分；不要全量套模板。
   - 若上游登记了重复主体、成组记录、前后测量或时间层级，验证记录必须写明划分单位、组边界和信息截止点；不得把同一组的相关记录分到训练与验证两侧。组标识不足时，将结论标为受限并按反馈策略处理。
3. 将每个 `out-*` 绑定到真实 `exp-*`、文件路径和 SHA-256。
4. 在 `results-manifest.yaml` 只写由结果或文献支持的主张，并附范围和限制。
5. 生成 `reproducibility.json`，实际复核至少一条核心复现命令。
6. 需要改变题意或模型时生成 `modeling-feedback.yaml`；不要用文字限制掩盖应当修订的失败。
7. 运行校验器并报告 `ready_for_writing`。

图表服务于判断和后续论证，不按固定数量生成，也不在这里撰写论文。
