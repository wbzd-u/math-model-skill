# 论文写作工作流

1. 写一句核心论证：在什么问题中，用什么模型，得到什么经过验证的回答，边界在哪里。
2. 建立术语与符号台账，锁定名称、符号、单位和缩写。
3. 建立段落任务图：背景、问题、方法、结果、比较、解释、限制各自分开。
4. 从证据向外起草。模型和结果附近保留来源映射，不把证据集中堆在结尾。
5. 先完成问题回答、模型、求解、结果与评价，再收束摘要和结论。
6. 逐项核对正文数字、公式、图表、引用与 `source-map.yaml`。
7. 按官方模板渲染，检查页码、图表题注、公式、交叉引用和溢出。

修改时优先修正被指出的主张、段落或结构，不整篇重写已正确部分。术语台账只有在明确改名时才更新。
