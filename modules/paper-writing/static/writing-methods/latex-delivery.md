# 第六轮：LaTeX 正式排版与可复现交付

除非当届官方规则或用户提供的模板明确要求 DOCX，正式稿默认采用 LaTeX。Markdown 稿继续作为可审计的内容主稿；`manuscript.tex` 是正式排版源，二者的结论、公式、图表和引用键必须一致。

## 源文件布局

```text
paper/
  manuscript.md
  manuscript.tex
  sections/                 # 篇幅或协作需要时拆分为 .tex 文件
  figures/                  # 仅引用 solver 产出的真实图表
  references.bib
  build/                     # 编译产生，不作为唯一交付证据
  manuscript.pdf
```

按任务需要拆分章节，不能为了形式强制拆分。不要将 YAML、临时日志、源数据或内部路径写入论文正文。

## 实施规则

1. 先读取官方规则，确定提交格式、语言、页数、匿名要求和模板限制。
2. 用当前赛题的官方模板或用户认可的最小模板建立 `manuscript.tex`；不从优秀论文复制导言区、正文、图表或参考文献组合。
3. 中文或混排时优先使用与可用 TeX 发行版兼容的 Unicode/CJK 路线；英文稿不为追求统一而引入无用宏包。
4. 公式符号、表格数值、图表文件、引用键和结论必须回链到 `terminology-ledger.yaml`、`source-map.yaml` 和 solver 产物。
5. 使用 `latex-compile` 编译实际主文件；保留编译命令、引擎、日志、警告和生成 PDF 的哈希。
6. 对 PDF 进行页面级检查：公式、图表、表格、页码、浮动体、参考文献、溢出文字和空白页。修复后重新编译，不以“成功生成 PDF”代替视觉校验。

## 格式例外

官方 DOCX 模板、指定在线系统或用户明确要求 DOCX 时，DOCX 是正式源。此时仍可用 LaTeX 作为内部草稿或公式验证工具，但不得把转换造成的格式问题转嫁给提交件。

没有可用 TeX 发行版时，报告环境缺口并按 `latex-doctor` 或运行时安装流程处理；在工具缺失时不伪称已经生成或检查 PDF。
