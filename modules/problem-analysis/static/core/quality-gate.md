# 题目分析阶段门禁

## 必须通过

- 所有输入文件已登记且原始文件保持只读。
- 比赛、分析范围和规则来源已确认或明确标记未知。
- 每项题目要求都有来源锚点和分类。
- 关键实体和变量已登记角色、单位、粒度和来源。
- 每个子问题都有输入、输出、目标或待回答结论。
- 约束和评价标准没有遗漏。
- 每个实质子问题都有可检查的语义契约、成功判定和验收测试。
- 动态、耦合、信息时点、聚合或定义难点已证据化，或已说明无实质难点的依据。
- 数据字段、单位、粒度和主要质量问题已登记。
- 每个子问题在数据任务矩阵中有对应记录。
- 子问题依赖图中的节点和边均可解析。
- 事实、歧义和假设分开记录。
- 结构化文件之间的 ID 引用一致。
- 未输出具体模型、算法或求解器选择。
- 十二项对抗性审查均已执行，且没有 `FAIL`。

## 阻断条件

- 题目正文或关键附件缺失。
- 关键字段含义无法确认，且会改变子问题定义。
- 子问题的输出要求无法从题面确定。
- 官方规则与题面要求冲突但未解决。
- 出现无法解释的循环依赖。
- 为推进流程而虚构数据、条件或评价指标。
- 未解决的歧义会改变目标、硬约束、成功判定、聚合口径、核心实体或输出接口。

## 门禁结论

只能使用：

- `PASS`：结构化交接包完整，可进入建模选择。
- `PASS_WITH_OPEN_ITEMS`：仅保留不改变问题定义的精度、格式或非关键数据风险；不得包含未解决的定义性歧义。
- `BLOCKED`：缺失信息会改变问题定义，必须先补充或确认。

报告必须列出结论、依据、未完成项和下一阶段不得忽略的风险。
