# 结构压力测试协议

对每个实质子问题，基于已登记证据写入 `semantic_contract` 和 `difficulty_drivers`。不要复述题面、推荐模型或输出隐藏推理过程。

1. **最小闭环**：写清“外生/给定量与可控量 → 状态或事件变化 → 成功判定 → 评价聚合 → 必交输出”。任一缺口登记为歧义或数据缺口。
2. **成败谓词**：明确何时算成功、有效、可行或更优；说明它是点值、区间、累计量、并集、概率、排序或多目标聚合。
3. **结构难点**：最多记录三项定义、动态、耦合、信息时点、聚合、尺度、不确定性或范围难点。每项必须说明它会改变 `objective`、`feasible_set`、`evaluation`、`data`、`output` 或 `dependency` 中的至少一项。
4. **挑战**：对高影响定义给出竞争性解释；对动态或资源约束检查零值、上限、同时发生、重叠、并列、不可行或缺失信息等边界。
5. **裁决**：每项结论必须链接证据，并归为事实、歧义、假设或下游待处理项。无法确认时不得补全。

若没有实质难点，使用 `difficulty_assessment.status: none-evidenced` 并说明检查依据；不要为了满足字段而制造伪难点。
