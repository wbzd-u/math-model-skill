# 第二轮：实体与变量分析

## 目标

识别研究对象、属性、关系和量纲，建立题目语义骨架。此轮描述变量角色，不定义数学模型。

## 必须执行

1. 列出人、地点、设备、资源、事件、时间段、网络节点或其他研究实体。
2. 为每个量标记角色：`given`、`unknown`、`controllable`、`state`、`outcome` 或 `index`。
3. 记录符号（如题面已有）、单位、取值范围、时间粒度、空间粒度和来源。
4. 标记实体之间的物理、时间、空间、包含、流动或决策关系。
5. 对同名异义、异名同义和单位冲突建立歧义项。
6. 区分外生/给定量、可控量、状态量和结果量，为后续 `semantic_contract` 提供变量 ID；识别事件及其发生时点。

## 提示问题

- 什么对象在变化，什么对象保持固定？
- 哪些量由题目给定，哪些量需要估计或决定？
- 哪些量会评价答案好坏？
- 变量是单值、时间序列、空间场、矩阵、图还是类别？
- 题目是否混用了不同单位、尺度或统计口径？
- 哪些量在多个子问题中必须保持同一含义？
- 哪些量一经设定就约束后续事件或多个决策？

## 输出

填写 `entity-variable-map.yaml`，并同步更新 `ambiguity-register.yaml` 和 `assumption-register.yaml`。

## 失败信号

- 变量没有单位或粒度且未登记未知原因。
- 把可控量和观测量混为一类。
- 为变量直接绑定算法、损失函数或求解器。
- 后续子问题使用了未在实体变量表中登记的核心量。
