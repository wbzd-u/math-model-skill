# 第四轮：数据与任务对齐

## 目标

验证每个子问题是否有足够、适配且可追溯的数据输入，明确数据缺口对问题定义的影响。

## 必须执行

1. 完成数据文件、工作表、字段、单位、粒度、时间和空间范围审计。
2. 为每个子问题建立一条数据任务矩阵记录。
3. 区分 `available`、`derivable`、`external-required`、`assumption-required` 和 `missing`。
4. 记录字段质量、关联键、样本覆盖和标签泄漏风险。
5. 只描述缺口，不在本阶段选择插补、标准化、特征工程或预测方法。
6. 核对每个子问题的成功判定和信息时点所需观测是否可用，防止把未来信息或未提供字段带入定义。

## 提示问题

- 这个子问题需要哪些观测、参数和评价基准？
- 哪些字段直接可用，哪些需要从现有字段推导？
- 数据粒度和任务粒度是否一致？
- 多个表如何连接，连接是否会造成重复或丢失？
- 缺失数据会阻止定义问题，还是只影响后续模型精度？
- 外部数据是否被题目允许，来源和时间范围如何约束？

## 输出

更新 `data-inventory.yaml` 和 `data-task-matrix.yaml`；将阻断缺口写入 `ambiguity-register.yaml`。

## 失败信号

- 数据清单与子问题之间没有映射。
- 默认所有字段都可直接用于分析。
- 忽略单位、时间范围或表连接粒度。
- 把“可能可以推导”写成“已有数据”。
