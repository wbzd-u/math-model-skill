# 第八轮：结构判断提示

在已有题面、附件和歧义登记的证据范围内，逐项思考下列问题。它们用于改善问题定义，不用于推荐算法或制造隐藏难点。

1. 每个子问题最终必须回答哪一种结论：数值、方案、排序、概率、机制解释、预测、分类还是可行性判断？若不先写清结论，后续方法如何被验收？
2. 题面中的对象、记录、字段或样例是否只是同一真实实体的局部观测？哪些 ID、时间、空间或单位证据支持或否定这种对应关系？
3. 哪个词、单位、附件字段、模板列或例外规则最可能改变目标、约束、状态、信息时点或输出接口？忽略它会怎样误解题目？
4. 多个小问是独立任务，还是前一问在新增条件下的扩展？如果是扩展，新增的到底是数据、机制、约束、评价还是交付物？
5. 用最简单的事实闭环重述本题：给定什么、可控制什么、什么状态/事件发生、何为成功、如何评价、必须交付什么？缺少哪一环就登记为歧义或数据缺口。

只有能回链到题面、附件、官方规则或明确标记为推断的问题才写入产物。若这些问题没有揭示实质结构，记录“未发现额外结构”，不要为了显得深入而编造难点。
